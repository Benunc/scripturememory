# Version 1.7.1 - Invite Flow: Verse Set Onboarding Fix

This patch release fixes onboarding for invite links that include a verse set (and optionally a group code), ensuring the correct join/apply dialog is reachable and that the invite context is preserved across magic-link verification.

## Bug Fixes

### Invite Link Onboarding
- Made the `/invite` route public so `https://scripture.wpsteward.com/invite?verseSet=...` no longer redirects to sign-in for logged-out users
- Preserved `verseSet` and `groupCode` across magic-link verification by returning users to their original invite URL
- Added an authenticated “Apply Invite” experience on `/invite` so users can add the verse set and/or join the group after login

### Backend API Support
- Added `POST /auth/join-group` to support joining an invited group directly from the invite page (authenticated + Turnstile protected)

## Migration Notes

### No Breaking Changes
- No database schema changes were introduced by this patch
- No additional migrations are required

### Deployment Order
- Deploy the worker changes before the frontend so the new authenticated invite actions work immediately
