# Version 2.0.0 - Family Games: Collaborative Scripture Memorization

A major new feature that transforms Scripture Memory into a collaborative, family-friendly game experience. Family Games allows authenticated users to create multiplayer memorization challenges where family members can join using a simple game code and compete together in real-time.

## 🎮 What's New

### **Family Games System**
- **Game creation**: Authenticated users can create multiplayer games with customizable rounds
- **Simple joining**: Family members join with just a 6-character game code and display name
- **No authentication required**: Participants don't need accounts to join games
- **Real-time gameplay**: Multiple devices can participate simultaneously in the same game

### **Game Structure**
- **Multiple rounds**: Each game consists of user-selectable rounds (verses)
- **Customizable difficulty**: Creators choose which words to hide in each verse
- **Time limits**: Configurable time limit per round (default 5 minutes)
- **Round synchronization**: All participants must finish before advancing to next round

### **Gameplay Features**
- **Word selection interface**: Participants select words from a multiple-choice list
- **Visual feedback**: Color-coded buttons (green for correct, yellow for wrong position, red for not in verse)
- **Progress tracking**: Real-time display of words completed, points earned, and current streak
- **Verse display**: Verse shown with blanks for hidden words, filled in as correct selections are made
- **Minimum 10 options**: Always shows at least 10 word options (or 1.5x hidden words if more)
- **Smart distractors**: Mix of Bible words and family-friendly silly words as incorrect options

### **Scoring System**
- **Accuracy-based scoring**: Rewards correct guesses with points
- **Speed bonus**: Faster correct selections earn more points
- **Streak rewards**: Consecutive correct guesses build streaks for bonus points
- **Leaderboard**: Real-time rankings showing points, rounds completed, accuracy, and streaks
- **Round statistics**: Detailed stats per round and overall game performance

### **Creator Controls**
- **Game management**: Start games, approve participants, open rounds, end games
- **Participant approval**: Manual or automatic approval of joining participants
- **Display name selection**: Creators choose their display name during game creation
- **Round management**: Open rounds when all participants are ready
- **Soft disconnect**: Mark participants as disconnected if they leave mid-game

### **User Experience**
- **Mobile-optimized**: Designed specifically for phone screens with compact layouts
- **No scrolling required**: All gameplay elements fit on a single screen
- **Clear visual hierarchy**: Easy to see verse, options, and progress
- **Responsive design**: Works seamlessly on all device sizes
- **Dark mode support**: Full support for light and dark themes

## 🔧 Technical Implementation

### **Backend Architecture**
- **New database tables**: 7 new tables for games, rounds, participants, progress, selections, and statistics
- **Polling-based updates**: Efficient state synchronization without WebSockets
- **Rate limiting**: Server-side protection against rapid word selections (100ms minimum)
- **Atomic operations**: Database transactions ensure data consistency
- **Automatic cleanup**: Cron job cleans up expired games hourly

### **API Endpoints**
- **POST /family-games/create**: Create a new game (authenticated)
- **POST /family-games/:gameCode/join**: Join a game with display name
- **GET /family-games/:gameCode/state**: Get current game state
- **POST /family-games/:gameCode/rounds/:roundNumber/open**: Open a round (creator)
- **POST /family-games/:gameCode/rounds/:roundNumber/start**: Start a round (participant)
- **POST /family-games/:gameCode/rounds/:roundNumber/select-word**: Select a word
- **POST /family-games/:gameCode/start**: Start the game (creator)
- **POST /family-games/:gameCode/end**: End the game (creator)

### **Frontend Components**
- **GameCreator**: Interface for creating games with rounds and word selection
- **GameJoiner**: Simple join interface for participants
- **GamePlay**: Main gameplay interface with dynamic state management
- **useFamilyGame**: Custom React hook for game state management and polling

### **Database Schema**
- **family_games**: Core game information and status
- **family_game_rounds**: Verse data and round configuration
- **family_game_participants**: Participant information and approval status
- **family_game_round_progress**: Individual progress per round
- **family_game_selections**: Word selection history for scoring
- **family_game_stats**: Overall game statistics and leaderboard
- **family_game_round_stats**: Per-round statistics

### **Word Distractor System**
- **Bible words**: Curated lists of nouns, verbs, adjectives, and adverbs from scripture
- **Silly words**: Family-friendly fun words for incorrect options
- **50/50 mix**: Distractors split evenly between Bible and silly words
- **Verse exclusion**: Never shows visible verse words as options (only hidden words)
- **Smart generation**: Ensures minimum options and proper word variety

## 🎯 Game Flow

### **For Game Creators**
1. **Create game**: Select display name, add rounds (verses), choose words to hide
2. **Share code**: Get 6-character game code to share with family
3. **Approve participants**: Review and approve joining family members (or auto-approve)
4. **Start game**: Begin the game when ready
5. **Open rounds**: Open each round when all participants are ready
6. **View leaderboard**: See real-time rankings between rounds
7. **End game**: Complete the game and view final results

### **For Participants**
1. **Join game**: Enter game code and choose display name
2. **Wait for approval**: Get approved by creator (if manual approval enabled)
3. **Wait for start**: Game creator starts the game
4. **Start round**: Click "Start Round" when round is opened
5. **Select words**: Choose words from options to fill in blanks
6. **Track progress**: See points, streak, and words completed
7. **Finish round**: Complete all words or time runs out
8. **View leaderboard**: See rankings between rounds
9. **Continue**: Proceed to next round when opened

## 📊 Scoring Details

### **Point Calculation**
- **Base points**: Points for each correct word selection
- **Speed bonus**: Faster selections earn additional points
- **Streak bonus**: Consecutive correct guesses multiply points
- **Accuracy tracking**: Percentage of correct vs incorrect selections

### **Leaderboard Metrics**
- **Total points**: Sum of all points earned across rounds
- **Rounds completed**: Number of rounds finished
- **Words completed**: Total words correctly selected
- **Longest streak**: Best consecutive correct selections
- **Accuracy percentage**: Overall correctness rate
- **Rank**: Position based on total points and accuracy

## 🔒 Privacy & Security

### **Data Isolation**
- **Separate tables**: Family games data completely isolated from user verses
- **No cross-contamination**: Game data doesn't affect personal memorization progress
- **Temporary data**: Games expire after 1 hour and are automatically cleaned up
- **Participant anonymity**: Participants only identified by display names

### **Access Control**
- **Creator authentication**: Only authenticated users can create games
- **Participant approval**: Creators control who can join (manual or auto)
- **Game code security**: 6-character codes are unique per active game
- **Rate limiting**: Protection against abuse with server-side limits

## 🚀 Performance

### **Optimizations**
- **Efficient polling**: Creators poll every 5 seconds, participants every 10 seconds (only between rounds)
- **No polling during gameplay**: Participants see only their own progress during active rounds
- **Database indexing**: Optimized queries for fast game state retrieval
- **Batch operations**: Atomic database transactions for consistency
- **Client-side timers**: Local device timers reduce server load

### **Scalability**
- **Stateless design**: No server-side session management required
- **UUID participants**: Unique identifiers without database lookups
- **Efficient queries**: Optimized database schema with proper indexes
- **Automatic cleanup**: Expired games cleaned up automatically
